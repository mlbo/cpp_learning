# cpp_learning 文档重构与逐日改写方案（评审稿）

> 目标：先给你一份可评审的“改造蓝图”，确认后再逐天重写，不直接改原教程内容。

## 1. 现状诊断（基于仓库扫描）

### 1.1 已确认问题

1. 35 天文档都有，但风格不统一，篇幅跨度很大（约 132 行到 907 行）。
2. 同类章节命名不一致：`学习目标/今日概览/知识点一/Part 1/...` 混用，新手难形成稳定阅读习惯。
3. EMC++（你说的 EFC 这里按 EMC++ 处理）在不少天里偏“结论罗列”，缺少“为什么”和“反例-正例对照”。
4. 算法讲解深度不稳定：有些天代码多但思维模板少，有些天有模板但边界条件不足。
5. “学完如何自检”不够标准化，复盘闭环不统一。
6. 仓库存在两套文档路径：`week_xx/day_xx/README.md` 为源，`docs/content/...` 是同步发布副本（由 `docs/sync_from_repo.sh` 生成）。

### 1.2 改造原则

1. 不改学习路线（仍是 35 天），只提升讲解质量和统一性。
2. 每天都要有“入门解释 + 工程视角 + 面试视角 + 自测闭环”。
3. EMC++ 每个条款都按同一深度模板重讲。
4. 每天保持固定阅读节奏，让新手形成“看一眼目录就知道怎么学”的肌肉记忆。

## 2. 统一文档模板（所有 Day README 强制统一）

> 每天 README 统一为以下 12 段结构（允许内容长短不同，不允许段落顺序乱）。

```markdown
# Day XX｜主题名

## A. 今天你将掌握什么（3-5条，可验证）
## B. 先修检查（昨天知识回顾 + 5分钟热身）
## C. 概念总览（一句话定义 + 生活类比 + 工程价值）
## D. 知识点1：数据结构/算法
## E. 知识点2：C++语言/并发/底层机制
## F. EMC++ 深入（条款精讲，固定 7 步法）
## G. LeetCode 双题精讲（讲解题 + 练习题）
## H. 代码导读（按本日 code 目录逐文件说明）
## I. 常见错误与调试清单（最少 5 条）
## J. 今日自测（概念题 + 手写题 + 复杂度题）
## K. 运行方式与预期输出
## L. 总结与明日衔接（下一天预热）
```

### 2.1 格式统一细则

1. 标题统一：`# Day XX｜中文主题`。
2. 二级标题固定为 `A-L`，防止结构漂移。
3. 每天必须出现：复杂度分析、边界条件、至少 1 个错误示例。
4. 每天必须有“术语小词典”（中英对照，5-10 个词）。
5. 每天必须有“本日最小可运行命令”（一键运行 + 手动运行两种）。

## 3. EMC++ 深化标准（所有出现条款的天都按此重写）

> 你提到“条款太简单看完不深入”，核心就改这里。之后每个条款不再只写结论，统一使用“7 步讲透法”。

### 3.1 条款 7 步讲透法（固定模板）

1. 问题场景：先给一个真实会踩坑的业务片段。
2. 错误写法：展示“能编译但有风险”的代码。
3. 条款原意：用一句人话解释作者到底在防什么坑。
4. 底层机制：类型推导/对象生命周期/并发语义等为什么导致问题。
5. 正确写法：给出改进后的可运行版本。
6. 边界与例外：什么时候可以不按条款做，代价是什么。
7. 面试追问：准备 2-3 个高频追问与答题框架。

### 3.2 EMC++ 写作要求

1. 每个条款至少 1 组“反例 vs 正例”。
2. 每个条款至少 1 个“工程实践建议”（代码评审规则、命名、API 设计）。
3. 每个条款至少 1 个“与 LeetCode/数据结构题的连接点”。

2. 禁止只做措辞润色，不补结构与深度。
3. 禁止引入仓库中不存在的代码文件路径。
4. 禁止把 EMC++ 变成条款口号，不给反例与机制。
5. 禁止改成“百科式大段堆砌”，必须服务新手上手。

### 8.6 建议给执行模型的提示词（可直接粘贴）

```markdown
你现在是 cpp_learning 仓库的“教学文档重构执行者”。请严格执行以下规则：

【目标】
- 把 Day 文档改造成新手可系统学习的统一教程。
- 强化 EMC++ 条款深度：每个条款都要有反例、正例、机制解释、边界条件。

【硬约束】
1. 仅修改 week_xx/day_xx/README.md（必要时可改 week_xx/README.md 索引）。
2. 不允许修改任何 .cpp/.h/CMake/sh 代码文件。
3. docs/content 是同步产物，不直接编辑。
4. 每天必须使用 A-L 结构模板，不得删节结构。

【执行顺序】
- 先只改 Day 1，输出后等待确认。
- 确认后每批最多改 3 天，并给出每批变更摘要和自检结果。

【每天必做】
- 读取当天 code 目录，保证文档中的代码路径真实存在。
- EMC++ 按“7 步讲透法”写。
- 增加常见错误清单（>=5）和今日自测（>=6题）。

【输出格式】
1. 修改了哪些文件
2. 本批每天的关键增强点
3. 自检结果（按验收清单逐条）
4. 未解决风险（如有）
```
