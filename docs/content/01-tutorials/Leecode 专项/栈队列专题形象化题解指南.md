## 📚 Day 15: 有效的括号 - 形象化讲解

### 🎮 题目理解（生活类比）

想象你在整理一堆盘子：

```
输入: "{[()]}"

想象成叠盘子:
┌─────────────┐
│    ( )      │  ← 最里面的盘子
│   [ ]       │  ← 中间的盘子  
│  { }        │  ← 最外面的盘子
└─────────────┘

规则: 后放进去的盘子必须先拿出来（后进先出）
```

### 🧠 核心思想：栈的匹配

#### 形象比喻：配对游戏

```
括号配对规则:
( 和 ) 是一对
[ 和 ] 是一对
{ 和 } 是一对

就像情侣配对:
- 左括号是"男"
- 右括号是"女"
- 每个右括号必须找到最近的未配对左括号

栈就是一个"等待配对"的候车室
```

#### 动画演示

```
输入: "{[()]}"

Step 1: 遇到 { (左括号，入栈)
栈: [{]

Step 2: 遇到 [ (左括号，入栈)
栈: [{, []

Step 3: 遇到 ( (左括号，入栈)
栈: [{, [, (]

Step 4: 遇到 ) (右括号，检查栈顶)
栈顶是 ( ✓ 配对成功！弹出
栈: [{, []

Step 5: 遇到 ] (右括号，检查栈顶)
栈顶是 [ ✓ 配对成功！弹出
栈: [{]

Step 6: 遇到 } (右括号，检查栈顶)
栈顶是 { ✓ 配对成功！弹出
栈: [] (空)

最终栈为空 → 有效字符串！
```

#### 错误情况示例

```
输入: "{[)]}"

Step 1-3: 同上，栈: [{, [, (]

Step 4: 遇到 ) (右括号，检查栈顶)
栈顶是 ( ✓ 配对成功！弹出
栈: [{, []

Step 5: 遇到 ] (右括号，检查栈顶)
栈顶是 [ ✓ 配对成功！弹出
栈: [{]

Step 6: 遇到 } (右括号，检查栈顶)
栈顶是 { ✓ 配对成功！弹出
栈: []

但此时还有字符！这就是问题所在...
```

#### 记忆口诀

```
括号匹配三步走:
1. 左括号，直接入栈
2. 右括号，看栈顶配对
3. 最后栈空才算对
```

## 📚 Day 17: 每日温度 - 单调栈经典题

### 🎮 题目理解（生活类比）

```
每日温度: [73, 74, 75, 71, 69, 72, 76, 73]
问: 每天要等几天才会更暖和？

想象你在排队等热奶茶:
Day 0: 73度 → 要等1天才能喝到74度的
Day 1: 74度 → 要等1天才能喝到75度的
Day 2: 75度 → 要等4天才能喝到76度的
...
```

### 🧠 单调栈思想

#### 形象比喻：递减队列

```
想象一群人按温度排队:
- 如果你比前面的人温度高，前面的人就知道答案了
- 前面的人离开队列，你继续往前看

栈维护一个"还没找到答案"的日期列表
```

#### 动画演示

```
温度: [73, 74, 75, 71, 69, 72, 76, 73]
      [ 0,  1,  2,  3,  4,  5,  6,  7]  ← 索引

=== Day 0: 温度73 ===
栈为空，0入栈
栈: [0]

=== Day 1: 温度74 ===
74 > 73 (栈顶温度)
→ Day 0 找到答案: 1 - 0 = 1 天后更暖和
→ 弹出0，继续检查（栈空）
栈: []，1入栈
栈: [1]
答案: [1, 0, 0, 0, 0, 0, 0, 0]

=== Day 2: 温度75 ===
75 > 74 (栈顶温度)
→ Day 1 找到答案: 2 - 1 = 1 天后更暖和
→ 弹出1，继续检查（栈空）
栈: []，2入栈
栈: [2]
答案: [1, 1, 0, 0, 0, 0, 0, 0]

=== Day 3: 温度71 ===
71 < 75 (栈顶温度)，不处理
栈: [2, 3]
答案: [1, 1, 0, 0, 0, 0, 0, 0]

=== Day 4: 温度69 ===
69 < 71 (栈顶温度)，不处理
栈: [2, 3, 4]
答案: [1, 1, 0, 0, 0, 0, 0, 0]

=== Day 5: 温度72 ===
72 > 69 (栈顶温度)
→ Day 4 找到答案: 5 - 4 = 1 天后更暖和
→ 弹出4，继续检查
72 > 71 (栈顶温度)
→ Day 3 找到答案: 5 - 3 = 2 天后更暖和
→ 弹出3，继续检查
72 < 75 (栈顶温度)，停止
栈: [2, 5]
答案: [1, 1, 0, 2, 1, 0, 0, 0]

=== Day 6: 温度76 ===
76 > 72 (栈顶温度)
→ Day 5 找到答案: 6 - 5 = 1 天后更暖和
→ 弹出5，继续检查
76 > 75 (栈顶温度)
→ Day 2 找到答案: 6 - 2 = 4 天后更暖和
→ 弹出2，栈空
栈: []，6入栈
栈: [6]
答案: [1, 1, 4, 2, 1, 1, 0, 0]

=== Day 7: 温度73 ===
73 < 76 (栈顶温度)，不处理
栈: [6, 7]

最终答案: [1, 1, 4, 2, 1, 1, 0, 0]
```

#### 单调栈核心要点

```
单调栈: 栈内元素保持单调性（递增或递减）

本题使用单调递减栈:
- 栈内存储"还没找到答案"的索引
- 栈顶到栈底，温度递增（栈顶温度最低）
- 当遇到更高温度时，弹出栈顶并计算答案

时间复杂度: O(n) - 每个元素最多入栈出栈各一次
空间复杂度: O(n) - 栈最多存储n个元素
```

## 📚 Day 18: 柱状图最大矩形 - 单调栈进阶

### 🎮 题目理解（生活类比）

```
柱状图高度: [2, 1, 5, 6, 2, 3]

想象成城市天际线:
    █
  █ █
  █ █
  █ █ █
█ █ █ █ █
2 1 5 6 2 3

问: 能框出的最大矩形面积是多少？
```

### 🧠 核心思想

#### 形象比喻：扩展法

```
对于每根柱子，想象它能向左右扩展多远:
- 向左扩展，直到遇到比它矮的柱子
- 向右扩展，直到遇到比它矮的柱子
- 面积 = 高度 × 宽度

关键: 如何快速找到左右边界？→ 单调栈！
```

#### 动画演示

```
高度: [2, 1, 5, 6, 2, 3]
      [0, 1, 2, 3, 4, 5]  ← 索引

我们维护一个单调递增栈（存储索引）

=== 处理索引0, 高度2 ===
栈空，0入栈
栈: [0]

=== 处理索引1, 高度1 ===
1 < 2，触发计算:
- 弹出0，高度=2
- 左边界=-1（栈空），右边界=1
- 宽度=1-(-1)-1=1，面积=2×1=2
栈: []，1入栈
栈: [1]

=== 处理索引2, 高度5 ===
5 > 1，入栈
栈: [1, 2]

=== 处理索引3, 高度6 ===
6 > 5，入栈
栈: [1, 2, 3]

=== 处理索引4, 高度2 ===
2 < 6，触发计算:
- 弹出3，高度=6
- 左边界=2，右边界=4
- 宽度=4-2-1=1，面积=6×1=6
栈: [1, 2]

2 < 5，继续触发:
- 弹出2，高度=5
- 左边界=1，右边界=4
- 宽度=4-1-1=2，面积=5×2=10
栈: [1]

2 > 1，停止
栈: [1, 4]

=== 处理索引5, 高度3 ===
3 > 2，入栈
栈: [1, 4, 5]

=== 结束，处理剩余元素 ===
弹出5: 高度=3, 左=4, 右=6, 宽度=6-4-1=1, 面积=3
弹出4: 高度=2, 左=1, 右=6, 宽度=6-1-1=4, 面积=8
弹出1: 高度=1, 左=-1, 右=6, 宽度=6-(-1)-1=6, 面积=6

最大面积: 10
```

## 📚 Day 19: 前K个高频元素 - 堆的应用

### 🎮 题目理解（生活类比）

```
数组: [1,1,1,2,2,3], k=2
问: 出现频率前2高的元素是哪些？

想象投票:
候选人: [1号, 2号, 3号]
得票数: [3票, 2票, 1票]

排名: 1号第1名(3票), 2号第2名(2票), 3号第3名(1票)
前2名: [1号, 2号] → [1, 2]
```

### 🧠 小顶堆思想

#### 形象比喻：K个座位

```
想象一个只有K个座位的VIP室:
- 当前K个元素先进入
- 新来的元素和最弱的比
- 如果新来的更强，踢掉最弱的

堆顶就是"最弱的VIP"，随时准备被替换
```

#### 动画演示

```
频率统计: {1:3, 2:2, 3:1}
K=2，维护大小为2的小顶堆

Step 1: 加入(1, 3)
堆: [(1,3)]
堆顶: (1,3)

Step 2: 加入(2, 2)
堆: [(1,3), (2,2)]
调整后（按频率排序）: [(2,2), (1,3)]
堆顶: (2,2)，频率最低

Step 3: 加入(3, 1)
频率1 < 堆顶频率2，跳过（不够格进VIP）

最终堆内: [(2,2), (1,3)]
结果: [1, 2] 或 [2, 1]
```

#### 为什么用小顶堆？

```
堆的大小固定为K:
- 大顶堆: 要知道最小的K个，但我们需要弹最大的，不方便
- 小顶堆: 堆顶就是最小的，正好判断新元素是否应该替换

时间复杂度: O(n log k)
- 建堆: O(n)
- n次堆操作: n × O(log k) = O(n log k)
```

## 📚 Day 20: 二叉树层序遍历 - BFS入门

### 🎮 题目理解（生活类比）

```
二叉树:
        3
       / \
      9  20
        /  \
       15   7

层序遍历: [[3], [9,20], [15,7]]

想象公司组织架构:
        CEO(3)
       /      \
   经理(9)   经理(20)
             /    \
          员工(15) 员工(7)

按层级访问: CEO → 经理们 → 员工们
```

### 🧠 BFS队列思想

#### 形象比喻：排队办事

```
队列就像办事窗口的排队:
- 先来的人先办（FIFO）
- 每层的人一起办，办完后叫下一层的人

队列就是"等待处理的名单"
```

#### 动画演示

```
树:     3
       / \
      9  20
        /  \
       15   7

=== 初始 ===
队列: [3]
结果: []

=== 第1轮 ===
取出3，记录
队列: []
结果: [[3]]
把3的子节点加入队列
队列: [9, 20]

=== 第2轮 ===
本层大小=2，处理2个节点
取出9，记录，无子节点
取出20，记录
队列: []
结果: [[3], [9, 20]]
把20的子节点加入队列
队列: [15, 7]

=== 第3轮 ===
本层大小=2，处理2个节点
取出15，记录，无子节点
取出7，记录，无子节点
队列: []
结果: [[3], [9, 20], [15, 7]]

队列空，结束
```

## 📚 Day 21: 最小栈 - 双栈设计

### 🎮 题目理解

```
实现一个栈，要在O(1)时间内获取最小值

普通栈:  [5, 2, 3, 1, 4]
最小值:  [5, 2, 2, 1, 1]  ← 每个位置对应的最小值

用两个栈:
- 主栈: 存储实际数据
- 辅助栈: 存储当前最小值
```

### 🧠 双栈设计

#### 形象比喻：记分牌

```
想象比赛记分牌:
- 主栈是选手得分
- 辅助栈是当前最高分（或最低分）

每次入栈，同时更新记录
每次出栈，同时删除记录
```

#### 动画演示

```
操作序列: push(5), push(2), push(3), push(1), getMin(), pop(), getMin()

=== push(5) ===
主栈: [5]
辅助栈: [5]  ← 当前最小5

=== push(2) ===
主栈: [5, 2]
辅助栈: [5, 2]  ← 2比5小，更新最小值为2

=== push(3) ===
主栈: [5, 2, 3]
辅助栈: [5, 2, 2]  ← 3比2大，最小值还是2

=== push(1) ===
主栈: [5, 2, 3, 1]
辅助栈: [5, 2, 2, 1]  ← 1比2小，更新最小值为1

=== getMin() ===
返回辅助栈顶: 1

=== pop() ===
主栈: [5, 2, 3]
辅助栈: [5, 2, 2]  ← 同时弹出

=== getMin() ===
返回辅助栈顶: 2
```

## 💡 栈队列学习技巧

### 栈的应用场景

```
1. 括号匹配、表达式求值
2. 单调栈：找下一个更大/更小元素
3. DFS深度优先搜索
4. 函数调用栈
```

### 队列的应用场景

```
1. BFS广度优先搜索
2. 层序遍历
3. 滑动窗口问题
4. 任务调度
```

### 单调栈模板

```cpp
vector<int> nextGreater(vector<int>& nums) {
    int n = nums.size();
    vector<int> result(n, -1);
    stack<int> st;  // 存储索引
    
    for (int i = 0; i < n; ++i) {
        while (!st.empty() && nums[i] > nums[st.top()]) {
            result[st.top()] = i;
            st.pop();
        }
        st.push(i);
    }
    return result;
}
```

### BFS模板

```cpp
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> result;
    queue<TreeNode*> q;
    if (root) q.push(root);
    
    while (!q.empty()) {
        int size = q.size();  // 当前层大小
        vector<int> level;
        for (int i = 0; i < size; ++i) {
            TreeNode* node = q.front(); q.pop();
            level.push_back(node->val);
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        result.push_back(level);
    }
    return result;
}
```
