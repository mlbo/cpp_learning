## 📚 Day 29: 二叉树遍历 - 树的基础

### 🎮 题目理解（生活类比）

```
二叉树:
        1
       / \
      2   3
     / \
    4   5

三种遍历方式:
- 前序: 根→左→右 → 1,2,4,5,3
- 中序: 左→根→右 → 4,2,5,1,3
- 后序: 左→右→根 → 4,5,2,3,1
```

### 🧠 遍历思想

#### 形象比喻：家谱访问

```
想象家谱树:
        爷爷(1)
       /      \
    爸爸(2)   叔叔(3)
    /    \
 哥哥(4) 我(5)

前序（先拜访长辈）: 爷爷→爸爸→哥哥→我→叔叔
中序（左到右）: 哥哥→爸爸→我→爷爷→叔叔
后序（先拜访晚辈）: 哥哥→我→爸爸→叔叔→爷爷
```

#### 动画演示（前序遍历）

```
树:     1
       / \
      2   3
     / \
    4   5

递归调用栈:

visit(1):
  ├─ 访问1 ✓
  ├─ visit(2):
  │    ├─ 访问2 ✓
  │    ├─ visit(4):
  │    │    ├─ 访问4 ✓
  │    │    ├─ visit(nullptr) → 返回
  │    │    └─ visit(nullptr) → 返回
  │    └─ visit(5):
  │         ├─ 访问5 ✓
  │         └─ ...
  └─ visit(3):
       ├─ 访问3 ✓
       └─ ...

结果: [1, 2, 4, 5, 3]
```

#### 迭代法（使用栈）

```
前序遍历迭代版:

初始: root=1, 栈=[]

Step 1: 访问1，右子3入栈，左子2入栈
栈: [3, 2]  ← 2在栈顶（后进先出）
结果: [1]

Step 2: 弹出2，访问2，右子5入栈，左子4入栈
栈: [3, 5, 4]
结果: [1, 2]

Step 3: 弹出4，访问4
栈: [3, 5]
结果: [1, 2, 4]

Step 4: 弹出5，访问5
栈: [3]
结果: [1, 2, 4, 5]

Step 5: 弹出3，访问3
栈: []
结果: [1, 2, 4, 5, 3]
```

#### 遍历模板

```cpp
// 前序递归
void preorder(TreeNode* root, vector<int>& res) {
    if (!root) return;
    res.push_back(root->val);  // 访问根
    preorder(root->left, res);  // 遍历左
    preorder(root->right, res); // 遍历右
}

// 前序迭代
vector<int> preorderTraversal(TreeNode* root) {
    vector<int> res;
    stack<TreeNode*> st;
    if (root) st.push(root);
    
    while (!st.empty()) {
        TreeNode* node = st.top(); st.pop();
        res.push_back(node->val);
        if (node->right) st.push(node->right);  // 右先入栈
        if (node->left) st.push(node->left);    // 左后入栈
    }
    return res;
}
```

## 📚 Day 30: 层序遍历 - BFS应用

### 🎮 题目理解

```
二叉树层序遍历:
        3
       / \
      9  20
        /  \
       15   7

结果: [[3], [9, 20], [15, 7]]

每层一个数组，从上到下，从左到右
```

### 🧠 BFS队列思想

#### 形象比喻：楼层扫描

```
想象一栋楼:
3楼:     [3]
2楼:   [9, 20]
1楼: [15, 7]

从顶楼开始，一层层扫描
队列就像电梯，按顺序载人
```

#### 动画演示

```
树:     3
       / \
      9  20
        /  \
       15   7

队列: [3], 结果: []

=== 第1轮 ===
队列大小=1，处理1个节点
弹出3，访问，记录
队列: []
结果: [[3]]
3的子节点入队: 9, 20
队列: [9, 20]

=== 第2轮 ===
队列大小=2，处理2个节点
弹出9，访问，记录（无子节点）
弹出20，访问，记录
队列: []
结果: [[3], [9, 20]]
20的子节点入队: 15, 7
队列: [15, 7]

=== 第3轮 ===
队列大小=2，处理2个节点
弹出15，访问，记录（无子节点）
弹出7，访问，记录（无子节点）
队列: []
结果: [[3], [9, 20], [15, 7]]

队列空，结束
```

#### BFS模板

```cpp
vector<vector<int>> levelOrder(TreeNode* root) {
    vector<vector<int>> result;
    queue<TreeNode*> q;
    
    if (root) q.push(root);
    
    while (!q.empty()) {
        int levelSize = q.size();  // 当前层节点数
        vector<int> level;
        
        for (int i = 0; i < levelSize; ++i) {
            TreeNode* node = q.front(); q.pop();
            level.push_back(node->val);
            
            if (node->left) q.push(node->left);
            if (node->right) q.push(node->right);
        }
        
        result.push_back(level);
    }
    
    return result;
}
```

## 📚 Day 31: 验证二叉搜索树 - BST性质

### 🎮 题目理解

```
BST（二叉搜索树）性质:
- 左子树所有节点 < 根节点
- 右子树所有节点 > 根节点
- 左右子树也是BST

        5
       / \
      3   7
     / \ / \
    1  4 6  8

验证: 每个节点都在合法范围内
```

### 🧠 范围验证法

#### 形象比喻：上下级关系

```
想象公司层级:
- 你的值必须在上级规定的范围内
- 上级给你的下级规定更窄的范围

        5 (范围: -∞, +∞)
       /              \
      3               7
(范围: -∞, 5)    (范围: 5, +∞)
     / \
    1   4
(-∞,3) (3,5)
```

#### 动画演示

```
树:     5
       / \
      3   7
     / \
    1   4

验证过程:

validate(5, min=-∞, max=+∞):
  5在(-∞, +∞)范围内 ✓
  ├─ validate(3, min=-∞, max=5):
  │    3在(-∞, 5)范围内 ✓
  │    ├─ validate(1, min=-∞, max=3):
  │    │    1在(-∞, 3)范围内 ✓
  │    │    子树为空 → 返回true
  │    └─ validate(4, min=3, max=5):
  │         4在(3, 5)范围内 ✓
  │         子树为空 → 返回true
  └─ validate(7, min=5, max=+∞):
       7在(5, +∞)范围内 ✓
       子树为空 → 返回true

所有验证通过 → 是BST
```

#### 代码模板

```cpp
bool isValidBST(TreeNode* root) {
    return validate(root, LONG_MIN, LONG_MAX);
}

bool validate(TreeNode* node, long minVal, long maxVal) {
    if (!node) return true;
    
    // 检查当前节点是否在范围内
    if (node->val <= minVal || node->val >= maxVal) {
        return false;
    }
    
    // 递归检查左右子树
    return validate(node->left, minVal, node->val) &&
           validate(node->right, node->val, maxVal);
}
```

## 📚 Day 32: 二叉树最大深度 - DFS/BFS

### 🎮 题目理解

```
树的最大深度 = 从根到最远叶子的层数

        3
       / \
      9  20
        /  \
       15   7

深度: 3层 → 返回3
```

### 🧠 三种解法

#### 解法1: DFS递归

```
maxDepth(root) = 1 + max(maxDepth(left), maxDepth(right))

     3
    / \
   9  20
      / \
     15  7

计算过程:
maxDepth(3) = 1 + max(maxDepth(9), maxDepth(20))
            = 1 + max(1, 1 + max(maxDepth(15), maxDepth(7)))
            = 1 + max(1, 1 + max(1, 1))
            = 1 + max(1, 2)
            = 3
```

#### 解法2: BFS层序

```
统计层数:

队列: [3], 层数: 0

处理第1层:
队列: [9, 20], 层数: 1

处理第2层:
队列: [15, 7], 层数: 2

处理第3层:
队列: [], 层数: 3

返回3
```

#### 解法3: DFS迭代（栈）

```
栈元素: (节点, 深度)

初始: [(3, 1)]
最大深度: 0

弹出(3, 1):
最大深度 = max(0, 1) = 1
推入: (9, 2), (20, 2)
栈: [(9, 2), (20, 2)]

弹出(20, 2):
最大深度 = max(1, 2) = 2
推入: (15, 3), (7, 3)
栈: [(9, 2), (15, 3), (7, 3)]

...继续处理...

最终最大深度: 3
```

## 📚 Day 33: 二叉树路径 - 回溯入门

### 🎮 题目理解

```
找出所有从根到叶子的路径

        1
       / \
      2   3
       \
        5

路径: ["1->2->5", "1->3"]
```

### 🧠 回溯思想

#### 形象比喻：走迷宫

```
从入口(根)出发，记录走过的路
走到死胡同(叶子)，记录完整路径
后退，尝试其他岔路

关键: 记住走过的路，走完后要"擦掉"
```

#### 动画演示

```
树:     1
       / \
      2   3
       \
        5

=== DFS搜索 ===

进入节点1:
路径: [1]

进入节点2:
路径: [1, 2]

进入节点2的左子节点:
节点为空，返回

进入节点5:
路径: [1, 2, 5]

5是叶子节点:
记录路径: "1->2->5"
回溯，弹出5
路径: [1, 2]

回溯，弹出2
路径: [1]

进入节点3:
路径: [1, 3]

3是叶子节点:
记录路径: "1->3"
回溯，弹出3
路径: [1]

结束

结果: ["1->2->5", "1->3"]
```

#### 回溯模板

```cpp
vector<string> binaryTreePaths(TreeNode* root) {
    vector<string> result;
    vector<int> path;
    dfs(root, path, result);
    return result;
}

void dfs(TreeNode* node, vector<int>& path, vector<string>& result) {
    if (!node) return;
    
    path.push_back(node->val);  // 选择
    
    // 如果是叶子节点，记录路径
    if (!node->left && !node->right) {
        string s;
        for (int i = 0; i < path.size(); ++i) {
            s += to_string(path[i]);
            if (i < path.size() - 1) s += "->";
        }
        result.push_back(s);
    }
    
    dfs(node->left, path, result);
    dfs(node->right, path, result);
    
    path.pop_back();  // 撤销选择（回溯）
}
```

## 📚 Day 34: 最近公共祖先 - LCA问题

### 🎮 题目理解

```
找两个节点的最近公共祖先

        3
       / \
      5   1
     / \ / \
    6  2 0  8
      / \
     7   4

LCA(5, 1) = 3
LCA(5, 4) = 5
```

### 🧠 递归思想

#### 形象比喻：家谱查祖

```
两个人往上追溯祖先:
- 你爸和你的公共祖先是爷爷
- 你和你兄弟的公共祖先是爸爸
- 你和自己的公共祖先是你自己

递归查每个人是否包含目标节点
```

#### 动画演示

```
树:     3
       / \
      5   1
     / \
    6   2

找LCA(5, 1):

LCA(3):
  左子树(5)找到了p ✓
  右子树(1)找到了q ✓
  左右都找到 → 当前节点3就是LCA

找LCA(5, 4):

LCA(3):
  左子树找到了p(5)和q(4) ✓
  右子树什么都没找到
  → LCA在左子树

LCA(5):
  当前节点就是p → 返回5
  左子树找到了q(4)
  当前节点在pq路径上 → 当前节点5就是LCA
```

#### 代码模板

```cpp
TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
    // 基础情况
    if (!root || root == p || root == q) {
        return root;
    }
    
    // 在左右子树中查找
    TreeNode* left = lowestCommonAncestor(root->left, p, q);
    TreeNode* right = lowestCommonAncestor(root->right, p, q);
    
    // 左右都找到 → 当前节点是LCA
    if (left && right) return root;
    
    // 只有一边找到 → 返回找到的那边
    return left ? left : right;
}
```

## 📚 Day 35: 并发编程入门 - 线程安全

### 🎮 并发基础概念

```
并发问题: 多个线程同时访问共享数据

例子: 两人同时取钱
账户: 1000元
A取500，B取500
如果并发控制不当，可能都成功（余额变500而非0）
```

### 🧠 线程安全方案

#### 方案1: 互斥锁（Mutex）

```
互斥锁就像卫生间门锁:
- 一个人进去，锁门
- 其他人必须等待
- 出来后开锁，下一个人才能进

代码:
std::mutex mtx;
mtx.lock();    // 加锁
// 访问共享数据
mtx.unlock();  // 解锁
```

#### 方案2: RAII锁管理

```
使用lock_guard自动管理锁:

{
    std::lock_guard<std::mutex> lock(mtx);
    // 访问共享数据
}  // 自动解锁，即使发生异常

类比: 自动门，人走后自动关
```

#### 方案3: 原子操作

```
原子操作: 不可中断的单步操作

std::atomic<int> counter(0);
counter++;  // 原子递增，不需要锁

类比: 一个不可打断的动作
```

### 线程安全计数器示例

```cpp
#include <mutex>
#include <atomic>

// 方案1: 使用互斥锁
class SafeCounter {
public:
    void increment() {
        std::lock_guard<std::mutex> lock(mutex_);
        ++count_;
    }
    
    int get() {
        std::lock_guard<std::mutex> lock(mutex_);
        return count_;
    }
    
private:
    int count_ = 0;
    std::mutex mutex_;
};

// 方案2: 使用原子操作
class AtomicCounter {
public:
    void increment() {
        ++count_;  // 原子操作
    }
    
    int get() {
        return count_.load();
    }
    
private:
    std::atomic<int> count_{0};
};
```

## 📚 并发编程进阶 - 生产者消费者

### 🎮 问题理解

```
生产者: 生产数据放入缓冲区
消费者: 从缓冲区取出数据处理

问题:
- 缓冲区满了，生产者要等待
- 缓冲区空了，消费者要等待
- 需要线程安全的队列
```

### 🧠 条件变量

#### 形象比喻：餐厅等位

```
条件变量就像餐厅叫号:
- 没座位时，顾客等待（wait）
- 有座位时，服务员通知（notify）
- 被通知后，顾客醒来取号

std::condition_variable:
- wait(): 等待通知
- notify_one(): 通知一个等待者
- notify_all(): 通知所有等待者
```

#### 线程安全队列实现

```cpp
template<typename T>
class ThreadSafeQueue {
public:
    void push(T value) {
        {
            std::lock_guard<std::mutex> lock(mutex_);
            queue_.push(std::move(value));
        }
        cond_.notify_one();  // 通知消费者
    }
    
    T pop() {
        std::unique_lock<std::mutex> lock(mutex_);
        // 等待队列非空
        cond_.wait(lock, [this] { return !queue_.empty(); });
        T value = std::move(queue_.front());
        queue_.pop();
        return value;
    }
    
private:
    std::queue<T> queue_;
    std::mutex mutex_;
    std::condition_variable cond_;
};
```

## 💡 树与并发学习技巧

### 树的遍历总结

```
DFS（深度优先）:
- 递归: 代码简洁
- 迭代: 用栈模拟递归

BFS（广度优先）:
- 用队列实现
- 适合层序遍历、最短路径

选择:
- 深度相关问题 → DFS
- 层级相关问题 → BFS
```

### 并发编程要点

```
1. 识别共享数据
2. 选择同步机制:
   - 简单计数 → atomic
   - 复杂操作 → mutex
   - 等待条件 → condition_variable
   
3. 避免死锁:
   - 按固定顺序加锁
   - 使用scoped_lock
   - 尽量缩短锁的持有时间
```
