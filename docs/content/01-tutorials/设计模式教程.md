### 1.2 设计原则（SOLID）

| 原则 | 全称 | 含义 |
|------|------|------|
| **S** | Single Responsibility | 单一职责：一个类只负责一件事 |
| **O** | Open/Closed | 开闭原则：对扩展开放，对修改关闭 |
| **L** | Liskov Substitution | 里氏替换：子类可以替换父类 |
| **I** | Interface Segregation | 接口隔离：接口要小而专一 |
| **D** | Dependency Inversion | 依赖倒置：依赖抽象而非具体 |

## 2. 创建型模式

### 2.1 单例模式（面试必考）

**核心思想**：确保一个类只有一个实例，并提供全局访问点。

#### 懒汉式（线程不安全）

```cpp
class SingletonLazy {
public:
    static SingletonLazy* getInstance() {
        if (instance_ == nullptr) {
            instance_ = new SingletonLazy();
        }
        return instance_;
    }
    
    void doSomething() { std::cout << "Singleton working\n"; }
    
private:
    SingletonLazy() = default;
    ~SingletonLazy() = default;
    SingletonLazy(const SingletonLazy&) = delete;
    SingletonLazy& operator=(const SingletonLazy&) = delete;
    
    static SingletonLazy* instance_;
};

SingletonLazy* SingletonLazy::instance_ = nullptr;
```

#### 懒汉式（线程安全）

```cpp
#include <mutex>

class SingletonThreadSafe {
public:
    static SingletonThreadSafe* getInstance() {
        if (instance_ == nullptr) {  // 第一次检查
            std::lock_guard<std::mutex> lock(mutex_);
            if (instance_ == nullptr) {  // 第二次检查
                instance_ = new SingletonThreadSafe();
            }
        }
        return instance_;
    }
    
private:
    SingletonThreadSafe() = default;
    
    static SingletonThreadSafe* instance_;
    static std::mutex mutex_;
};
```

#### Meyers 单例（C++11 推荐）

```cpp
class SingletonMeyers {
public:
    static SingletonMeyers& getInstance() {
        static SingletonMeyers instance;  // C++11 保证线程安全
        return instance;
    }
    
    void doSomething() { std::cout << "Meyers Singleton\n"; }
    
private:
    SingletonMeyers() = default;
    ~SingletonMeyers() = default;
    SingletonMeyers(const SingletonMeyers&) = delete;
    SingletonMeyers& operator=(const SingletonMeyers&) = delete;
};

// 使用
// SingletonMeyers::getInstance().doSomething();
```

#### 面试要点

```
常见问题：
1. 单例模式的线程安全实现？
   答：使用 Meyers 单例或双重检查锁定

2. 单例模式的内存泄漏问题？
   答：Meyers 单例自动释放，或使用智能指针

3. 如何防止反射破坏单例？
   答：C++ 无反射机制，主要防止拷贝构造

4. 单例模式的缺点？
   答：全局状态、难以测试、违反单一职责
```

### 2.2 工厂方法模式

**核心思想**：定义一个创建对象的接口，让子类决定实例化哪个类。

```cpp
#include <iostream>
#include <memory>
#include <string>

// 产品接口
class Product {
public:
    virtual ~Product() = default;
    virtual std::string getName() const = 0;
};

// 具体产品 A
class ConcreteProductA : public Product {
public:
    std::string getName() const override { return "Product A"; }
};

// 具体产品 B
class ConcreteProductB : public Product {
public:
    std::string getName() const override { return "Product B"; }
};

// 工厂接口
class Factory {
public:
    virtual ~Factory() = default;
    virtual std::unique_ptr<Product> createProduct() = 0;
};

// 具体工厂 A
class ConcreteFactoryA : public Factory {
public:
    std::unique_ptr<Product> createProduct() override {
        return std::make_unique<ConcreteProductA>();
    }
};

// 具体工厂 B
class ConcreteFactoryB : public Factory {
public:
    std::unique_ptr<Product> createProduct() override {
        return std::make_unique<ConcreteProductB>();
    }
};

// 使用示例
void factoryDemo() {
    std::unique_ptr<Factory> factoryA = std::make_unique<ConcreteFactoryA>();
    auto productA = factoryA->createProduct();
    std::cout << productA->getName() << "\n";  // Product A
    
    std::unique_ptr<Factory> factoryB = std::make_unique<ConcreteFactoryB>();
    auto productB = factoryB->createProduct();
    std::cout << productB->getName() << "\n";  // Product B
}
```

#### 简单工厂（变体）

```cpp
enum class ProductType { A, B };

class SimpleFactory {
public:
    static std::unique_ptr<Product> create(ProductType type) {
        switch (type) {
            case ProductType::A:
                return std::make_unique<ConcreteProductA>();
            case ProductType::B:
                return std::make_unique<ConcreteProductB>();
        }
        return nullptr;
    }
};

// 使用
// auto product = SimpleFactory::create(ProductType::A);
```

### 2.3 抽象工厂模式

**核心思想**：创建相关或依赖对象的家族，而无需指定具体类。

```cpp
#include <iostream>
#include <memory>
#include <string>

// 产品家族：按钮
class Button {
public:
    virtual ~Button() = default;
    virtual void render() = 0;
};

class WindowsButton : public Button {
public:
    void render() override { std::cout << "Windows Button\n"; }
};

class MacOSButton : public Button {
public:
    void render() override { std::cout << "macOS Button\n"; }
};

// 产品家族：文本框
class TextBox {
public:
    virtual ~TextBox() = default;
    virtual void render() = 0;
};

class WindowsTextBox : public TextBox {
public:
    void render() override { std::cout << "Windows TextBox\n"; }
};

class MacOSTextBox : public TextBox {
public:
    void render() override { std::cout << "macOS TextBox\n"; }
};

// 抽象工厂
class GUIFactory {
public:
    virtual ~GUIFactory() = default;
    virtual std::unique_ptr<Button> createButton() = 0;
    virtual std::unique_ptr<TextBox> createTextBox() = 0;
};

// Windows 工厂
class WindowsFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<WindowsButton>();
    }
    std::unique_ptr<TextBox> createTextBox() override {
        return std::make_unique<WindowsTextBox>();
    }
};

// macOS 工厂
class MacOSFactory : public GUIFactory {
public:
    std::unique_ptr<Button> createButton() override {
        return std::make_unique<MacOSButton>();
    }
    std::unique_ptr<TextBox> createTextBox() override {
        return std::make_unique<MacOSTextBox>();
    }
};

// 客户端代码
void renderUI(GUIFactory& factory) {
    auto button = factory.createButton();
    auto textBox = factory.createTextBox();
    
    button->render();
    textBox->render();
}
```

### 2.4 建造者模式

**核心思想**：将复杂对象的构建与表示分离，使得同样的构建过程可以创建不同的表示。

```cpp
#include <iostream>
#include <memory>
#include <string>

// 复杂产品
class Computer {
public:
    std::string cpu;
    std::string ram;
    std::string storage;
    std::string gpu;
    
    void show() {
        std::cout << "Computer: CPU=" << cpu 
                  << ", RAM=" << ram
                  << ", Storage=" << storage
                  << ", GPU=" << gpu << "\n";
    }
};

// 建造者
class ComputerBuilder {
public:
    ComputerBuilder& setCPU(const std::string& cpu) {
        computer_.cpu = cpu;
        return *this;
    }
    
    ComputerBuilder& setRAM(const std::string& ram) {
        computer_.ram = ram;
        return *this;
    }
    
    ComputerBuilder& setStorage(const std::string& storage) {
        computer_.storage = storage;
        return *this;
    }
    
    ComputerBuilder& setGPU(const std::string& gpu) {
        computer_.gpu = gpu;
        return *this;
    }
    
    Computer build() { return std::move(computer_); }
    
private:
    Computer computer_;
};

// 使用（链式调用）
void builderDemo() {
    Computer computer = ComputerBuilder()
        .setCPU("Intel i9")
        .setRAM("32GB DDR5")
        .setStorage("1TB NVMe SSD")
        .setGPU("NVIDIA RTX 4090")
        .build();
    
    computer.show();
}
```

### 2.5 原型模式

**核心思想**：通过复制现有对象来创建新对象。

```cpp
#include <iostream>
#include <memory>
#include <string>

class Prototype {
public:
    virtual ~Prototype() = default;
    virtual std::unique_ptr<Prototype> clone() const = 0;
    virtual void show() const = 0;
};

class ConcretePrototype : public Prototype {
public:
    ConcretePrototype(const std::string& name, int value) 
        : name_(name), value_(value) {}
    
    // 拷贝构造实现克隆
    std::unique_ptr<Prototype> clone() const override {
        return std::make_unique<ConcretePrototype>(*this);
    }
    
    void show() const override {
        std::cout << "Prototype: " << name_ << ", " << value_ << "\n";
    }
    
private:
    std::string name_;
    int value_;
};

void prototypeDemo() {
    auto original = std::make_unique<ConcretePrototype>("Original", 100);
    auto clone = original->clone();
    
    original->show();
    clone->show();
}
```

## 3. 结构型模式

### 3.1 适配器模式（面试高频）

**核心思想**：将一个类的接口转换成客户期望的另一个接口。

```cpp
#include <iostream>
#include <memory>

// 旧接口
class OldSystem {
public:
    void oldMethod() {
        std::cout << "Old system method\n";
    }
};

// 新接口
class NewInterface {
public:
    virtual ~NewInterface() = default;
    virtual void newMethod() = 0;
};

// 适配器（类适配器：多重继承）
class ClassAdapter : public OldSystem, public NewInterface {
public:
    void newMethod() override {
        oldMethod();  // 调用旧方法
    }
};

// 适配器（对象适配器：组合）
class ObjectAdapter : public NewInterface {
public:
    ObjectAdapter(std::unique_ptr<OldSystem> oldSystem)
        : oldSystem_(std::move(oldSystem)) {}
    
    void newMethod() override {
        oldSystem_->oldMethod();
    }
    
private:
    std::unique_ptr<OldSystem> oldSystem_;
};

void adapterDemo() {
    // 类适配器
    NewInterface* adapter1 = new ClassAdapter();
    adapter1->newMethod();
    delete adapter1;
    
    // 对象适配器
    auto oldSystem = std::make_unique<OldSystem>();
    NewInterface* adapter2 = new ObjectAdapter(std::move(oldSystem));
    adapter2->newMethod();
    delete adapter2;
}
```

#### 面试要点

```
类适配器 vs 对象适配器：

类适配器：
- 使用多重继承
- 无法适配子类
- 效率稍高

对象适配器：
- 使用组合
- 可以适配子类
- 更灵活（推荐）
```

### 3.2 装饰器模式

**核心思想**：动态地给对象添加额外的职责。

```cpp
#include <iostream>
#include <memory>
#include <string>

// 组件接口
class Coffee {
public:
    virtual ~Coffee() = default;
    virtual std::string getDescription() const = 0;
    virtual double getCost() const = 0;
};

// 基础组件
class SimpleCoffee : public Coffee {
public:
    std::string getDescription() const override { return "Simple coffee"; }
    double getCost() const override { return 10.0; }
};

// 装饰器基类
class CoffeeDecorator : public Coffee {
public:
    CoffeeDecorator(std::unique_ptr<Coffee> coffee)
        : coffee_(std::move(coffee)) {}
    
protected:
    std::unique_ptr<Coffee> coffee_;
};

// 具体装饰器：牛奶
class MilkDecorator : public CoffeeDecorator {
public:
    using CoffeeDecorator::CoffeeDecorator;
    
    std::string getDescription() const override {
        return coffee_->getDescription() + ", milk";
    }
    
    double getCost() const override {
        return coffee_->getCost() + 2.0;
    }
};

// 具体装饰器：糖
class SugarDecorator : public CoffeeDecorator {
public:
    using CoffeeDecorator::CoffeeDecorator;
    
    std::string getDescription() const override {
        return coffee_->getDescription() + ", sugar";
    }
    
    double getCost() const override {
        return coffee_->getCost() + 1.0;
    }
};

void decoratorDemo() {
    auto coffee = std::make_unique<SimpleCoffee>();
    coffee = std::make_unique<MilkDecorator>(std::move(coffee));
    coffee = std::make_unique<SugarDecorator>(std::move(coffee));
    
    std::cout << coffee->getDescription() << "\n";  // Simple coffee, milk, sugar
    std::cout << "Cost: " << coffee->getCost() << "\n";  // 13
}
```

### 3.3 代理模式

**核心思想**：为其他对象提供代理以控制对这个对象的访问。

```cpp
#include <iostream>
#include <memory>
#include <string>

// 主题接口
class Image {
public:
    virtual ~Image() = default;
    virtual void display() = 0;
};

// 真实主题
class RealImage : public Image {
public:
    RealImage(const std::string& filename) : filename_(filename) {
        loadFromDisk();
    }
    
    void display() override {
        std::cout << "Displaying " << filename_ << "\n";
    }
    
private:
    void loadFromDisk() {
        std::cout << "Loading " << filename_ << " from disk\n";
    }
    
    std::string filename_;
};

// 代理
class ImageProxy : public Image {
public:
    ImageProxy(const std::string& filename) : filename_(filename) {}
    
    void display() override {
        if (!realImage_) {
            realImage_ = std::make_unique<RealImage>(filename_);
        }
        realImage_->display();
    }
    
private:
    std::string filename_;
    std::unique_ptr<RealImage> realImage_;
};

void proxyDemo() {
    // 使用代理延迟加载
    ImageProxy proxy("photo.jpg");
    
    std::cout << "Image created but not loaded yet\n";
    
    // 真正使用时才加载
    proxy.display();
    proxy.display();  // 第二次不会重新加载
}
```

#### 代理类型

```
代理模式分类：

1. 虚拟代理：延迟加载（如上例）
2. 保护代理：控制访问权限
3. 远程代理：代表远程对象
4. 智能引用：自动管理资源
```

### 3.4 外观模式

**核心思想**：为子系统中的一组接口提供一个一致的界面。

```cpp
#include <iostream>
#include <string>

// 子系统
class CPU {
public:
    void start() { std::cout << "CPU started\n"; }
    void shutdown() { std::cout << "CPU shutdown\n"; }
};

class Memory {
public:
    void load() { std::cout << "Memory loaded\n"; }
    void free() { std::cout << "Memory freed\n"; }
};

class HardDrive {
public:
    void read() { std::cout << "HardDrive reading\n"; }
    void stop() { std::cout << "HardDrive stopped\n"; }
};

// 外观
class ComputerFacade {
public:
    void start() {
        std::cout << "Starting computer...\n";
        cpu_.start();
        memory_.load();
        hardDrive_.read();
        std::cout << "Computer started\n";
    }
    
    void shutdown() {
        std::cout << "Shutting down computer...\n";
        hardDrive_.stop();
        memory_.free();
        cpu_.shutdown();
        std::cout << "Computer shutdown\n";
    }
    
private:
    CPU cpu_;
    Memory memory_;
    HardDrive hardDrive_;
};

void facadeDemo() {
    ComputerFacade computer;
    computer.start();
    computer.shutdown();
}
```

### 3.5 桥接模式

**核心思想**：将抽象部分与实现部分分离，使它们都可以独立变化。

```cpp
#include <iostream>
#include <memory>
#include <string>

// 实现接口
class Device {
public:
    virtual ~Device() = default;
    virtual bool isEnabled() const = 0;
    virtual void enable() = 0;
    virtual void disable() = 0;
    virtual int getVolume() const = 0;
    virtual void setVolume(int percent) = 0;
};

// 具体实现：电视
class TV : public Device {
public:
    bool isEnabled() const override { return on_; }
    void enable() override { on_ = true; std::cout << "TV on\n"; }
    void disable() override { on_ = false; std::cout << "TV off\n"; }
    int getVolume() const override { return volume_; }
    void setVolume(int percent) override { volume_ = percent; }
    
private:
    bool on_ = false;
    int volume_ = 30;
};

// 具体实现：收音机
class Radio : public Device {
public:
    bool isEnabled() const override { return on_; }
    void enable() override { on_ = true; std::cout << "Radio on\n"; }
    void disable() override { on_ = false; std::cout << "Radio off\n"; }
    int getVolume() const override { return volume_; }
    void setVolume(int percent) override { volume_ = percent; }
    
private:
    bool on_ = false;
    int volume_ = 20;
};

// 抽象
class RemoteControl {
public:
    RemoteControl(std::shared_ptr<Device> device) : device_(device) {}
    
    virtual void togglePower() {
        if (device_->isEnabled()) {
            device_->disable();
        } else {
            device_->enable();
        }
    }
    
    void volumeDown() {
        device_->setVolume(device_->getVolume() - 10);
    }
    
    void volumeUp() {
        device_->setVolume(device_->getVolume() + 10);
    }
    
protected:
    std::shared_ptr<Device> device_;
};

// 扩展抽象：高级遥控器
class AdvancedRemoteControl : public RemoteControl {
public:
    using RemoteControl::RemoteControl;
    
    void mute() {
        device_->setVolume(0);
        std::cout << "Muted\n";
    }
};

void bridgeDemo() {
    auto tv = std::make_shared<TV>();
    auto radio = std::make_shared<Radio>();
    
    RemoteControl tvRemote(tv);
    AdvancedRemoteControl radioRemote(radio);
    
    tvRemote.togglePower();
    radioRemote.togglePower();
    radioRemote.mute();
}
```

### 3.6 组合模式

**核心思想**：将对象组合成树形结构以表示"部分-整体"的层次结构。

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <string>

// 组件
class FileSystemComponent {
public:
    virtual ~FileSystemComponent() = default;
    virtual void display(int indent = 0) const = 0;
    virtual int getSize() const = 0;
};

// 叶子：文件
class File : public FileSystemComponent {
public:
    File(const std::string& name, int size) 
        : name_(name), size_(size) {}
    
    void display(int indent) const override {
        std::string padding(indent, ' ');
        std::cout << padding << "- " << name_ << " (" << size_ << "KB)\n";
    }
    
    int getSize() const override { return size_; }
    
private:
    std::string name_;
    int size_;
};

// 组合：文件夹
class Folder : public FileSystemComponent {
public:
    Folder(const std::string& name) : name_(name) {}
    
    void add(std::unique_ptr<FileSystemComponent> component) {
        children_.push_back(std::move(component));
    }
    
    void display(int indent) const override {
        std::string padding(indent, ' ');
        std::cout << padding << "+ " << name_ << "/\n";
        for (const auto& child : children_) {
            child->display(indent + 2);
        }
    }
    
    int getSize() const override {
        int total = 0;
        for (const auto& child : children_) {
            total += child->getSize();
        }
        return total;
    }
    
private:
    std::string name_;
    std::vector<std::unique_ptr<FileSystemComponent>> children_;
};

void compositeDemo() {
    auto root = std::make_unique<Folder>("root");
    
    auto folder1 = std::make_unique<Folder>("documents");
    folder1->add(std::make_unique<File>("resume.pdf", 100));
    folder1->add(std::make_unique<File>("notes.txt", 50));
    
    auto folder2 = std::make_unique<Folder>("pictures");
    folder2->add(std::make_unique<File>("photo1.jpg", 200));
    folder2->add(std::make_unique<File>("photo2.jpg", 300));
    
    root->add(std::move(folder1));
    root->add(std::move(folder2));
    root->add(std::make_unique<File>("readme.md", 10));
    
    root->display();
    std::cout << "Total size: " << root->getSize() << "KB\n";
}
```

### 3.7 享元模式

**核心思想**：运用共享技术有效地支持大量细粒度的对象。

```cpp
#include <iostream>
#include <memory>
#include <unordered_map>
#include <string>

// 享元：内在状态
class TreeType {
public:
    TreeType(const std::string& name, const std::string& color, 
             const std::string& texture)
        : name_(name), color_(color), texture_(texture) {}
    
    void draw(int x, int y) const {
        std::cout << "Drawing " << name_ << " (" << color_ << ", " 
                  << texture_ << ") at (" << x << ", " << y << ")\n";
    }
    
private:
    std::string name_;
    std::string color_;
    std::string texture_;
};

// 享元工厂
class TreeFactory {
public:
    static std::shared_ptr<TreeType> getTreeType(
        const std::string& name, 
        const std::string& color,
        const std::string& texture) 
    {
        std::string key = name + "_" + color + "_" + texture;
        
        auto it = treeTypes_.find(key);
        if (it != treeTypes_.end()) {
            return it->second;
        }
        
        auto treeType = std::make_shared<TreeType>(name, color, texture);
        treeTypes_[key] = treeType;
        return treeType;
    }
    
private:
    static std::unordered_map<std::string, std::shared_ptr<TreeType>> treeTypes_;
};

// 上下文：外在状态
class Tree {
public:
    Tree(int x, int y, std::shared_ptr<TreeType> type)
        : x_(x), y_(y), type_(type) {}
    
    void draw() const { type_->draw(x_, y_); }
    
private:
    int x_, y_;
    std::shared_ptr<TreeType> type_;
};

void flyweightDemo() {
    // 只创建三种树类型，但可以绘制大量树
    auto oakType = TreeFactory::getTreeType("Oak", "Green", "Rough");
    auto pineType = TreeFactory::getTreeType("Pine", "Dark Green", "Smooth");
    auto mapleType = TreeFactory::getTreeType("Maple", "Red", "Smooth");
    
    std::vector<Tree> forest;
    for (int i = 0; i < 5; ++i) {
        forest.emplace_back(i * 10, i * 20, oakType);
        forest.emplace_back(i * 10 + 5, i * 20 + 5, pineType);
        forest.emplace_back(i * 10 + 10, i * 20 + 10, mapleType);
    }
    
    for (const auto& tree : forest) {
        tree.draw();
    }
}
```

### 4.1 策略模式（面试高频）

**核心思想**：定义一系列算法，把它们封装起来，并使它们可以相互替换。

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <algorithm>
#include <functional>

// 策略接口
class SortStrategy {
public:
    virtual ~SortStrategy() = default;
    virtual void sort(std::vector<int>& data) = 0;
};

// 具体策略：快速排序
class QuickSort : public SortStrategy {
public:
    void sort(std::vector<int>& data) override {
        std::cout << "Quick sort\n";
        std::sort(data.begin(), data.end());
    }
};

// 具体策略：冒泡排序
class BubbleSort : public SortStrategy {
public:
    void sort(std::vector<int>& data) override {
        std::cout << "Bubble sort\n";
        for (size_t i = 0; i < data.size(); ++i) {
            for (size_t j = 0; j < data.size() - i - 1; ++j) {
                if (data[j] > data[j + 1]) {
                    std::swap(data[j], data[j + 1]);
                }
            }
        }
    }
};

// 上下文
class Sorter {
public:
    void setStrategy(std::unique_ptr<SortStrategy> strategy) {
        strategy_ = std::move(strategy);
    }
    
    void sort(std::vector<int>& data) {
        if (strategy_) {
            strategy_->sort(data);
        }
    }
    
private:
    std::unique_ptr<SortStrategy> strategy_;
};

// C++11 函数式实现
class SorterFunctional {
public:
    using SortFunction = std::function<void(std::vector<int>&)>;
    
    void setStrategy(SortFunction strategy) {
        strategy_ = std::move(strategy);
    }
    
    void sort(std::vector<int>& data) {
        if (strategy_) {
            strategy_(data);
        }
    }
    
private:
    SortFunction strategy_;
};

void strategyDemo() {
    std::vector<int> data = {5, 2, 8, 1, 9};
    
    Sorter sorter;
    sorter.setStrategy(std::make_unique<QuickSort>());
    sorter.sort(data);
    
    // 使用函数式
    SorterFunctional sorterFunc;
    sorterFunc.setStrategy([](std::vector<int>& d) {
        std::sort(d.begin(), d.end());
    });
    sorterFunc.sort(data);
}
```

### 4.2 观察者模式（面试高频）

**核心思想**：定义对象间的一对多依赖，当一个对象改变状态，所有依赖者都会收到通知。

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <string>
#include <functional>

// 观察者接口
class Observer {
public:
    virtual ~Observer() = default;
    virtual void update(const std::string& message) = 0;
};

// 具体观察者
class ConcreteObserver : public Observer {
public:
    ConcreteObserver(const std::string& name) : name_(name) {}
    
    void update(const std::string& message) override {
        std::cout << name_ << " received: " << message << "\n";
    }
    
private:
    std::string name_;
};

// 主题
class Subject {
public:
    void attach(std::shared_ptr<Observer> observer) {
        observers_.push_back(observer);
    }
    
    void detach(std::shared_ptr<Observer> observer) {
        observers_.erase(
            std::remove(observers_.begin(), observers_.end(), observer),
            observers_.end()
        );
    }
    
    void notify(const std::string& message) {
        for (auto& observer : observers_) {
            if (auto obs = observer.lock()) {
                obs->update(message);
            }
        }
    }
    
private:
    std::vector<std::weak_ptr<Observer>> observers_;
};

// C++11 函数式实现
class SubjectFunctional {
public:
    using Callback = std::function<void(const std::string&)>;
    
    size_t subscribe(Callback callback) {
        callbacks_.push_back(std::move(callback));
        return callbacks_.size() - 1;
    }
    
    void unsubscribe(size_t id) {
        callbacks_.erase(callbacks_.begin() + id);
    }
    
    void notify(const std::string& message) {
        for (const auto& callback : callbacks_) {
            callback(message);
        }
    }
    
private:
    std::vector<Callback> callbacks_;
};

void observerDemo() {
    Subject subject;
    
    auto observer1 = std::make_shared<ConcreteObserver>("Observer 1");
    auto observer2 = std::make_shared<ConcreteObserver>("Observer 2");
    
    subject.attach(observer1);
    subject.attach(observer2);
    
    subject.notify("Hello observers!");
    
    // 函数式版本
    SubjectFunctional subjFunc;
    int id = subjFunc.subscribe([](const std::string& msg) {
        std::cout << "Lambda received: " << msg << "\n";
    });
    
    subjFunc.notify("Hello lambda!");
}
```

### 4.3 命令模式

**核心思想**：将请求封装为对象，从而可用不同的请求对客户进行参数化。

```cpp
#include <iostream>
#include <memory>
#include <vector>
#include <string>
#include <stack>

// 接收者
class Light {
public:
    void on() { std::cout << "Light is ON\n"; }
    void off() { std::cout << "Light is OFF\n"; }
};

class TV {
public:
    void on() { std::cout << "TV is ON\n"; }
    void off() { std::cout << "TV is OFF\n"; }
};

// 命令接口
class Command {
public:
    virtual ~Command() = default;
    virtual void execute() = 0;
    virtual void undo() = 0;
};

// 具体命令
class LightOnCommand : public Command {
public:
    LightOnCommand(Light& light) : light_(light) {}
    
    void execute() override { light_.on(); }
    void undo() override { light_.off(); }
    
private:
    Light& light_;
};

class TVOnCommand : public Command {
public:
    TVOnCommand(TV& tv) : tv_(tv) {}
    
    void execute() override { tv_.on(); }
    void undo() override { tv_.off(); }
    
private:
    TV& tv_;
};

// 宏 命令
class MacroCommand : public Command {
public:
    MacroCommand(std::vector<std::unique_ptr<Command>> commands)
        : commands_(std::move(commands)) {}
    
    void execute() override {
        for (const auto& cmd : commands_) {
            cmd->execute();
        }
    }
    
    void undo() override {
        for (auto it = commands_.rbegin(); it != commands_.rend(); ++it) {
            (*it)->undo();
        }
    }
    
private:
    std::vector<std::unique_ptr<Command>> commands_;
};

// 调用者
class RemoteControl {
public:
    void setCommand(std::unique_ptr<Command> command) {
        command_ = std::move(command);
    }
    
    void pressButton() {
        if (command_) {
            command_->execute();
            history_.push(std::move(command_));
        }
    }
    
    void pressUndo() {
        if (!history_.empty()) {
            history_.top()->undo();
            history_.pop();
        }
    }
    
private:
    std::unique_ptr<Command> command_;
    std::stack<std::unique_ptr<Command>> history_;
};

void commandDemo() {
    Light light;
    TV tv;
    
    RemoteControl remote;
    
    remote.setCommand(std::make_unique<LightOnCommand>(light));
    remote.pressButton();
    
    remote.setCommand(std::make_unique<TVOnCommand>(tv));
    remote.pressButton();
    
    remote.pressUndo();
}
```

### 4.4 迭代器模式

**核心思想**：提供一种方法顺序访问聚合对象中的元素，而不暴露其内部表示。

```cpp
#include <iostream>
#include <vector>
#include <string>

template<typename T>
class Iterator {
public:
    virtual ~Iterator() = default;
    virtual bool hasNext() = 0;
    virtual T next() = 0;
};

template<typename T>
class Container {
public:
    virtual ~Container() = default;
    virtual std::unique_ptr<Iterator<T>> createIterator() = 0;
};

template<typename T>
class NameRepository : public Container<T> {
public:
    NameRepository(std::vector<T> names) : names_(std::move(names)) {}
    
    std::unique_ptr<Iterator<T>> createIterator() override {
        return std::make_unique<NameIterator>(this);
    }
    
    const std::vector<T>& getNames() const { return names_; }
    
private:
    std::vector<T> names_;
    
    class NameIterator : public Iterator<T> {
    public:
        NameIterator(NameRepository* repo) : repo_(repo) {}
        
        bool hasNext() override {
            return index_ < repo_->getNames().size();
        }
        
        T next() override {
            if (hasNext()) {
                return repo_->getNames()[index_++];
            }
            return T();
        }
        
    private:
        NameRepository* repo_;
        size_t index_ = 0;
    };
};

void iteratorDemo() {
    NameRepository<std::string> names({"Alice", "Bob", "Carol"});
    auto iter = names.createIterator();
    
    while (iter->hasNext()) {
        std::cout << iter->next() << "\n";
    }
}
```

### 4.6 责任链模式

**核心思想**：使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合。

```cpp
#include <iostream>
#include <memory>
#include <string>

class Handler {
public:
    virtual ~Handler() = default;
    
    Handler* setNext(Handler* next) {
        next_ = next;
        return next;
    }
    
    virtual std::string handle(const std::string& request) {
        if (next_) {
            return next_->handle(request);
        }
        return "Request unhandled";
    }
    
protected:
    Handler* next_ = nullptr;
};

class MonkeyHandler : public Handler {
public:
    std::string handle(const std::string& request) override {
        if (request == "Banana") {
            return "Monkey: I'll eat the " + request;
        }
        return Handler::handle(request);
    }
};

class SquirrelHandler : public Handler {
public:
    std::string handle(const std::string& request) override {
        if (request == "Nut") {
            return "Squirrel: I'll eat the " + request;
        }
        return Handler::handle(request);
    }
};

class DogHandler : public Handler {
public:
    std::string handle(const std::string& request) override {
        if (request == "Meat") {
            return "Dog: I'll eat the " + request;
        }
        return Handler::handle(request);
    }
};

void chainOfResponsibilityDemo() {
    MonkeyHandler monkey;
    SquirrelHandler squirrel;
    DogHandler dog;
    
    monkey.setNext(&squirrel)->setNext(&dog);
    
    std::cout << monkey.handle("Nut") << "\n";
    std::cout << monkey.handle("Banana") << "\n";
    std::cout << monkey.handle("Meat") << "\n";
    std::cout << monkey.handle("Coffee") << "\n";
}
```

### 4.7 状态模式

**核心思想**：允许对象在其内部状态改变时改变其行为。

```cpp
#include <iostream>
#include <memory>
#include <string>

class State;

class Context {
public:
    Context();
    
    void setState(std::unique_ptr<State> state);
    void request();
    
private:
    std::unique_ptr<State> state_;
};

class State {
public:
    virtual ~State() = default;
    virtual void handle(Context& context) = 0;
    virtual std::string name() const = 0;
};

class ConcreteStateA : public State {
public:
    void handle(Context& context) override;
    std::string name() const override { return "State A"; }
};

class ConcreteStateB : public State {
public:
    void handle(Context& context) override;
    std::string name() const override { return "State B"; }
};

Context::Context() : state_(std::make_unique<ConcreteStateA>()) {}

void Context::setState(std::unique_ptr<State> state) {
    std::cout << "Changing from " << state_->name() 
              << " to " << state->name() << "\n";
    state_ = std::move(state);
}

void Context::request() {
    state_->handle(*this);
}

void ConcreteStateA::handle(Context& context) {
    std::cout << "Handling in State A\n";
    context.setState(std::make_unique<ConcreteStateB>());
}

void ConcreteStateB::handle(Context& context) {
    std::cout << "Handling in State B\n";
    context.setState(std::make_unique<ConcreteStateA>());
}

void stateDemo() {
    Context context;
    
    context.request();  // A -> B
    context.request();  // B -> A
    context.request();  // A -> B
}
```

### 5.2 工厂模式 vs 抽象工厂

| 对比项 | 工厂方法 | 抽象工厂 |
|--------|----------|----------|
| 目的 | 创建一种产品 | 创建产品家族 |
| 新增产品 | 新增工厂类 | 修改工厂接口 |
| 新增产品族 | 无需修改 | 新增工厂类 |
| 复杂度 | 较低 | 较高 |

### 5.3 装饰器 vs 代理

| 对比项 | 装饰器 | 代理 |
|--------|--------|------|
| 目的 | 增加功能 | 控制访问 |
| 对象创建 | 客户端 | 代理内部 |
| 关注点 | 功能扩展 | 访问控制 |

### 5.4 策略模式 vs 状态模式

| 对比项 | 策略模式 | 状态模式 |
|--------|----------|----------|
| 目的 | 算法切换 | 状态转换 |
| 客户端 | 选择策略 | 不知道状态 |
| 行为 | 独立算法 | 状态相关 |
