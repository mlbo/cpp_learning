## 链表常见技巧汇总

### 1. 双指针技巧

| 技巧 | 应用场景 | 时间复杂度 |
|------|----------|------------|
| 快慢指针 | 找中点、检测环 | O(n) |
| 前后指针 | 删除倒数第k个节点 | O(n) |
| 交叉指针 | 找相交节点 | O(n+m) |

### 2. 虚拟头节点

```cpp
ListNode dummy(0);
dummy.next = head;
// 处理头节点删除等边界情况
return dummy.next;
```

**优点**：统一处理头节点边界情况，代码更简洁

### 3. 递归 vs 迭代

| 方式 | 优点 | 缺点 |
|------|------|------|
| 递归 | 代码简洁，思路清晰 | 栈空间 O(n) |
| 迭代 | 空间复杂度 O(1) | 代码稍复杂 |

## LeetCode 题目详解

### 160. 相交链表

**题目**：给你两个单链表的头节点 `headA` 和 `headB`，找出并返回两个单链表相交的起始节点。

**核心思路**：双指针交叉遍历
- 指针A遍历完链表A后，跳到链表B继续
- 指针B遍历完链表B后，跳到链表A继续
- 两指针最终会在相交点相遇（或同时到达末尾nullptr）

**关键公式**：
```
a + c + b = b + c + a
```
其中 a、b 是各自独立部分长度，c 是公共部分长度。

### 148. 排序链表

**题目**：给你链表的头节点 `head`，将其按升序排列并返回排序后的链表。

**要求**：O(n log n) 时间复杂度 + O(1) 空间复杂度

**核心思路**：归并排序
1. **找中点**：快慢指针找到链表中点
2. **分割**：从中间断开，递归排序两部分
3. **合并**：合并两个有序链表

**为什么选择归并排序？**
- 快速排序需要随机访问，链表不支持
- 归并排序适合链表的顺序访问特性
- 空间复杂度可优化为 O(1)
