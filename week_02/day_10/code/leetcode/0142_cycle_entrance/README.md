# LeetCode 142: 环形链表 II

## 题目描述

给定一个链表的头节点 `head`，返回链表开始入环的第一个节点。如果链表无环，则返回 `null`。

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 `pos` 是 `-1`，则在该链表中没有环。

**注意：** `pos` 不作为参数进行传递，仅仅是为了标识链表的实际情况。

**不允许修改链表。**

## 示例

```
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的链表节点
解释：链表中有一个环，其尾部连接到第二个节点。

输入：head = [1,2], pos = 0
输出：返回索引为 0 的链表节点
解释：链表中有一个环，其尾部连接到第一个节点。

输入：head = [1], pos = -1
输出：返回 null
解释：链表中没有环。
```

## 解法

### 方法一：快慢指针（推荐）

**核心思想：**
1. 快指针每次走两步，慢指针每次走一步
2. 如果有环，两指针必在环内某点相遇
3. 相遇后，将一个指针移回头部，两指针以相同速度前进
4. 再次相遇的点就是环入口

**数学推导：**

```
设：
- a: 头节点到入环点的距离
- b: 入环点到相遇点的距离
- c: 相遇点到入环点的距离

慢指针走的距离：a + b
快指针走的距离：a + n(b+c) + b

快指针是慢指针的两倍：
2(a + b) = a + (n+1)b + nc

推导：
2a + 2b = a + (n+1)b + nc
a = (n-1)(b+c) + c
```

**结论：** `a = (n-1)(b+c) + c`，说明从相遇点和头同时出发，必在入环点相遇！

```cpp
ListNode *detectCycle(ListNode *head) {
    ListNode *slow = head, *fast = head;
    
    // 找相遇点
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) break;
    }
    
    // 无环
    if (!fast || !fast->next) return nullptr;
    
    // 找入口
    slow = head;
    while (slow != fast) {
        slow = slow->next;
        fast = fast->next;
    }
    return slow;
}
```

**复杂度分析：**
- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 方法二：哈希表

使用哈希表记录访问过的节点，第一个重复访问的节点就是环入口。

```cpp
ListNode *detectCycle(ListNode *head) {
    unordered_set<ListNode*> visited;
    while (head) {
        if (visited.count(head)) return head;
        visited.insert(head);
        head = head->next;
    }
    return nullptr;
}
```

**复杂度分析：**
- 时间复杂度：O(n)
- 空间复杂度：O(n)

## 关键点

1. **快慢指针必相遇**：如果有环，快指针一定能追上慢指针
2. **相遇点不是入口**：需要进一步数学推导
3. **双指针同速前进**：从相遇点和头同时出发是关键技巧
4. **边界处理**：空链表、单节点、无环等特殊情况

## 延伸思考

1. 为什么快慢指针一定能相遇？
   - 环形跑道模型：快的人一定能追上慢的人

2. 为什么相遇后要一个指针回头部？
   - 数学证明 a = (n-1)(b+c) + c

3. 空间换时间的权衡
   - 哈希表方法更直观，但使用 O(n) 空间
   - 快慢指针需要数学推导，但只需 O(1) 空间
