## 📚 Day 8: 反转链表 - 形象化讲解

### 🎮 题目理解

想象一列火车，车厢之间用挂钩连接：

```
原火车:  🚃① → 🚃② → 🚃③ → 🚃④ → 🚃⑤ → 空地

目标: 让火车掉头
掉头后:  🚃⑤ → 🚃④ → 🚃③ → 🚃② → 🚃① → 空地
```

### 🧠 核心技巧：三指针法

```
三指针工作分配:

prev: 记录"已经反转好的部分"的头部
curr: 当前正在处理的节点
next: 记录下一个要处理的节点（防止丢失）

初始状态:
prev = null (还没有反转的部分)
curr = head (从头开始)
next = ? (待确定)

图示:
null    [1] → [2] → [3] → null
 prev    curr
          ↓
        next = curr.next (先记住下一个)
```

### 🎬 动画演示

```
=== 初始状态 ===
null    [1] → [2] → [3] → null
 prev    curr

=== Step 1: 记住下一个 ===
null    [1] → [2] → [3] → null
 prev    curr   next

=== Step 2: 反转当前指针 ===
null ← [1]    [2] → [3] → null
       curr   next
       (1.next = prev)

=== Step 3: 移动prev ===
null ← [1]    [2] → [3] → null
       prev   next
       (prev = curr)

=== Step 4: 移动curr ===
null ← [1]    [2] → [3] → null
       prev   curr
              (curr = next)

=== 下一轮 ===
null ← [1]    [2] → [3] → null
       prev   curr
              ↓
             next = curr.next

null ← [1] ← [2]    [3] → null
              curr   next

null ← [1] ← [2]    [3] → null
              prev   curr

=== 最终状态 ===
null ← [1] ← [2] ← [3]
                    prev (新头部!)
                    curr = null (循环结束)
```

### 💡 记忆口诀

```
反转链表四步走，记住口诀不迷路:

1. 存下个 (next = curr->next)  → 怕迷路，先记下
2. 反指针 (curr->next = prev)  → 让当前回头
3. 移前驱 (prev = curr)        → 前驱跟上
4. 移当前 (curr = next)        → 继续前进
```

### ⏱️ 复杂度分析

```
时间: O(n) - 每个节点只处理一次
空间: O(1) - 只用三个指针变量
```

## 📚 Day 8: 移除链表元素 - 形象化讲解

### 🎮 题目理解

```
链表: [1] → [2] → [6] → [3] → [6] → null
目标: 删除所有值为6的节点
结果: [1] → [2] → [3] → null

生活类比:
清理书架，把指定作者的书都拿走
```

### 🧠 核心技巧：虚拟头节点

**问题**: 如果要删除的是第一个节点怎么办？

```
无虚拟头节点的问题:
[6] → [2] → [3] → null
删除6时，head需要特殊处理！

有虚拟头节点:
[虚拟] → [6] → [2] → [3] → null
 dummy    head
删除6时，只需:
 dummy.next = head.next
 head = head.next
```

### 🎬 动画演示

```
=== 添加虚拟头节点 ===
[虚拟] → [1] → [2] → [6] → [3] → [6] → null
 dummy

=== 遍历过程 ===
Step 1: curr = dummy, curr.next = 1
        [虚拟] → [1] → [2] → [6] → [3] → [6]
         curr
        1 ≠ 6，curr后移

Step 2: curr = [1], curr.next = 2
        [虚拟] → [1] → [2] → [6] → [3] → [6]
                 curr
        2 ≠ 6，curr后移

Step 3: curr = [2], curr.next = 6
        [虚拟] → [1] → [2] → [6] → [3] → [6]
                          curr
        6 == 6！删除！
        curr.next = curr.next.next

删除后:
[虚拟] → [1] → [2] → [3] → [6] → null
                 curr

Step 4: curr = [2], curr.next = 3
        3 ≠ 6，curr后移

...继续到结束

=== 最终结果 ===
[虚拟] → [1] → [2] → [3] → null
返回: dummy.next
```

## 📚 Day 9: 环形链表 - 形象化讲解

### 🎮 题目理解

```
环形链表: 链表中有环，像操场跑道
[1] → [2] → [3] → [4] → [5]
              ↑___________↓

问题: 如何判断链表是否有环？
```

### 🧠 核心技巧：快慢指针（Floyd判圈算法）

**形象比喻**: 操场跑步

```
想象两个人在操场跑步:
- 慢跑者: 每次跑1步
- 快跑者: 每次跑2步

如果操场是直的:
快跑者会先到终点，两人永远不会相遇

如果操场是环形:
快跑者会追上慢跑者，两人会相遇！
```

### 🎬 动画演示

```
初始: slow和fast都在起点
[1] → [2] → [3] → [4] → [5] → [3] (回到3)
 ↑
slow,fast

Step 1: slow走1步, fast走2步
[1] → [2] → [3] → [4] → [5] → [3]
       ↑     ↑
      slow  fast

Step 2: slow走1步, fast走2步
[1] → [2] → [3] → [4] → [5] → [3]
             ↑           ↑
            slow        fast

Step 3: slow走1步, fast走2步
[1] → [2] → [3] → [4] → [5] → [3]
                   ↑     ↑
                  fast  slow

Step 4: slow走1步, fast走2步
[1] → [2] → [3] → [4] → [5] → [3]
             ↑
          slow,fast 相遇！有环！
```

### 💡 为什么一定能相遇？

```
数学证明（简化版）:

假设环长为C，快慢指针进入环后:
- 慢指针位置: p
- 快指针位置: (p+2) mod C

每次迭代，两者的距离缩小1:
距离: 0 → C-1 → C-2 → ... → 0 (相遇)

最多需要C次迭代就会相遇！
```

## 📚 Day 10: 找环入口 - 形象化讲解

### 🎮 题目理解

```
有环链表:
[1] → [2] → [3] → [4] → [5] → [6]
              ↑_______________↓
环入口是节点3

问题: 找到环的入口节点
```

### 🧠 核心技巧：两次相遇

```
第一阶段: 快慢指针相遇
第二阶段: 新指针从头走，慢指针继续走
两次相遇的点就是环入口！
```

### 🎬 动画演示

```
=== 第一阶段：快慢指针相遇 ===

初始: a=环外距离, b=环内相遇点距入口距离, C=环长

[1] → [2] → [3] → [4] → [5] → [6] → [3]...
 a      a     入口    b      b      b
慢指针走的距离: a + b
快指针走的距离: a + b + nC (n圈)

快指针速度是慢指针2倍:
2(a + b) = a + b + nC
=> a + b = nC
=> a = nC - b

=== 第二阶段：找入口 ===

从相遇点和起点同时出发，速度相同:
- 指针1从起点走a步，到达入口
- 指针2从相遇点走a步 = nC - b步
  = 绕了n圈少b步，刚好到达入口！

图示:
起点: [1] → [2] → [3]...
       ptr1         入口
                ↑
相遇点: ...[4] → [5] → [6] → [3]
              ptr2

ptr1走a步，ptr2走a步，在入口相遇！
```

### 💡 记忆口诀

```
找环入口两步走:
1. 快慢指针来相遇
2. 新指针对着走
   一个从头开始
   一个原地继续
   相遇就是入口！
```

## 📚 Day 11: 合并K个有序链表

### 🎮 题目理解

```
输入:
链表1: [1] → [4] → [5]
链表2: [1] → [3] → [4]
链表3: [2] → [6]

输出合并后:
[1] → [1] → [2] → [3] → [4] → [4] → [5] → [6]
```

### 🧠 方法一：优先队列（推荐）

**形象比喻**: 多队合一

```
想象有K个队伍，每队都按身高排好
要合并成一个大队伍，怎么做？

方法:
1. 从每个队头选最矮的人出来，排成一排
2. 每次选最矮的人进入大队伍
3. 该人后面的人补上空位

这就是"优先队列"的思想！
```

### 🎬 动画演示

```
=== 初始状态 ===
链表1: [1] → [4] → [5]
链表2: [1] → [3] → [4]
链表3: [2] → [6]

优先队列(最小堆):
┌───┐
│ 1 │ ← 链表1的头
│ 1 │ ← 链表2的头
│ 2 │ ← 链表3的头
└───┘

=== Step 1 ===
取出最小值1(来自链表1)
结果: [1]
链表1: [4] → [5]
优先队列: [1(链表2), 2(链表3), 4(链表1)]

=== Step 2 ===
取出最小值1(来自链表2)
结果: [1] → [1]
链表2: [3] → [4]
优先队列: [2(链表3), 3(链表2), 4(链表1)]

=== Step 3 ===
取出最小值2(来自链表3)
结果: [1] → [1] → [2]
链表3: [6]
优先队列: [3(链表2), 4(链表1), 6(链表3)]

...继续直到队列空...

=== 最终结果 ===
[1] → [1] → [2] → [3] → [4] → [4] → [5] → [6]
```

### 🧠 方法二：分治合并

```
分治思想:
合并K个 = 合并(前K/2个) + 合并(后K/2个)

图示:
[链表1] [链表2] [链表3] [链表4]
    ↓       ↓       ↓       ↓
  合并1&2  合并3&4
      ↓       ↓
    合并结果1&结果2
          ↓
       最终结果
```

## 💡 链表题目通用技巧

### 1. 虚拟头节点
```
什么时候用: 
- 可能需要删除头节点
- 需要统一处理逻辑

好处:
- 避免空指针判断
- 代码更简洁
```

### 2. 快慢指针
```
用途:
- 找链表中点
- 判断是否有环
- 找环入口
- 找倒数第k个节点

关键: slow走1步，fast走2步
```

### 3. 双指针
```
用途:
- 相交链表
- 删除倒数第N个

关键: 两个指针保持特定距离
```

### 4. 递归 vs 迭代
```
递归:
- 代码简洁
- 空间O(n)栈空间
- 可能栈溢出

迭代:
- 稍复杂
- 空间O(1)
- 更安全
```
